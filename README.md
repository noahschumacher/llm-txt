# llms.txt Generator

A web app that crawls any website and generates an [`llms.txt`](https://llmstxt.org) file — a proposed standard that helps LLMs understand and navigate site content, similar to how `robots.txt` guides crawlers.

Enter a URL, pick a mode, and get a ready-to-publish `llms.txt` in seconds. Generation progress streams live to the UI via SSE.

## Setup

**Prerequisites:** Go 1.22+

```bash
git clone https://github.com/noahschumacher/llm-txt
cd llm-txt

cp .env.example .env
# edit .env — at minimum set APP_ENV and APP_PORT

LLM_TXT_ENV_FILE=.env go run .
```

Open `http://localhost:8080`.

### Building

```bash
go build -o bin/llm-txt .
LLM_TXT_ENV_FILE=.env ./bin/llm-txt
```

The binary is fully self-contained — the frontend is embedded via `//go:embed`.

## Configuration

| Variable | Description | Required |
|---|---|---|
| `APP_ENV` | `local` \| `dev` \| `prod` | Yes |
| `APP_PORT` | Port to listen on | Yes |
| `LLM_PROVIDER` | `anthropic` \| `openai` | Enhanced mode only |
| `LLM_API_KEY` | API key for the LLM provider | Enhanced mode only |
| `LLM_MODEL` | Model to use (e.g. `claude-haiku-4-5`, `gpt-4o-mini`) | No |
| `CRAWL_MAX_PAGES` | Max pages to crawl per request (default: `50`) | No |
| `CRAWL_MAX_DEPTH` | Max BFS depth (default: `3`) | No |
| `CRAWL_DELAY_MS` | Delay between requests in ms (default: `500`) | No |
| `LLM_CONCURRENCY` | Max concurrent LLM calls (default: `5`) | No |

## Deployment (Sevalla)

1. Connect your GitHub repo in the Sevalla dashboard
2. Build command: `go build -o bin/llm-txt .`
3. Start command: `./bin/llm-txt`
4. Add environment variables under Settings → Environment Variables

Sevalla handles HTTPS termination — no additional config needed.

## How It Works

### Generation Modes

**Basic** — Uses page titles and `<meta name="description">` for descriptions. Fast, no API key required.

**Enhanced** — Same crawl, but descriptions are generated by an LLM from the page body content. Produces meaningfully better output. Requires a configured API key.

**Full Text** — Optional add-on to either mode. Also produces `llms-full.txt` with the full content of every crawled page concatenated under their section headers.

### Pipeline

1. Validate and normalize the URL
2. Fetch and parse `robots.txt` — build a disallow list respected by all subsequent requests
3. Discover `sitemap.xml` (follows `sitemap_index.xml` if present) to seed the crawl queue; falls back to BFS from the root
4. BFS crawl — up to `CRAWL_MAX_PAGES` pages at `CRAWL_MAX_DEPTH` levels deep; extracts `<title>`, `<meta name="description">`, `<h1>`, body text, and outbound links
5. Deduplicate and filter — strip fragments, normalize query params, drop pagination and tag pages
6. Infer sections from URL path segments (`/docs/*` → Docs, `/blog/*` → Blog, etc.)
7. Generate descriptions — meta description in basic mode; LLM-generated summaries in enhanced mode, dispatched concurrently via a bounded goroutine pool
8. Assemble `llms.txt` per spec: `H1` site name, blockquote description, `H2` sections, `[Title](url): Description` entries
9. Optionally assemble `llms-full.txt` with full page content under the same sections
10. Stream all progress events to the client via SSE; deliver final output when complete

### Architecture

Single Go binary — no separate frontend server or build step. The `static/` directory is embedded into the binary at compile time.

```
llm-txt/
├── main.go                      # env loading, dependency wiring, graceful shutdown
├── server/
│   ├── server.go                # server struct, route registration
│   ├── generate.go              # /generate handler, SSE streaming
│   ├── middleware/
│   │   ├── log.go               # request logging
│   │   └── timeout.go           # per-route timeout
│   └── services/
│       └── generator/
│           ├── service.go       # crawl → describe → format pipeline
│           ├── sections.go      # URL path → section inference
│           └── formatter.go     # llms.txt and llms-full.txt assembly
├── crawler/
│   ├── crawler.go               # BFS loop, depth/page limits, crawl delay
│   ├── robots.go                # robots.txt parsing
│   ├── sitemap.go               # sitemap.xml / sitemap_index.xml parsing
│   └── extractor.go             # HTML → title, meta, body, links
├── clients/
│   └── llm/
│       ├── client.go            # LLM API client, prompt construction
│       └── pool.go              # semaphore-based concurrent worker pool
├── pkg/
│   └── env.go                   # LoadStringEnv, LoadIntEnv, LoadBoolEnv, etc.
├── static/
│   └── index.html               # single-page frontend, vanilla JS
└── .env.example
```
